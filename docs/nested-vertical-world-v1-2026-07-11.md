# V1 · 纵向嵌套世界

## 目标

让悬浮岛屿的数量能够继续增长，同时保持世界可读。V1 不在默认产品中生成假问题岛，而是把真实问题岛组织为“世界／群岛 → 主岛 → 伴岛 → 岛内站点”的连续探索层级。

## 数据契约

- 每座岛仍是一个真实问题对象；默认数据仍为 27 座真实／样例岛。
- `altitudeZ` 是 0..1 的连续制图高度，由已有南北位置稳定折叠而来；`low / middle / high` 只是可读的三段名称。
- 每个计算群岛选择离区域几何中心最近的岛作为导航主岛，其余成员成为伴岛。
- 主岛只承担区域入口作用，与阶段、成员、活跃度、共识和研究价值无关。
- 岛间实线空路只来自已有账本 current；群岛归属不伪装成证据关系。

## 探索行为

1. 世界／区域尺度先显示群岛与导航主岛。
2. 镜头从中景靠近时，伴岛通过平滑 LOD 逐步显现。
3. 中景首次点击主岛只将镜头飞入近景，展开周围伴岛。
4. 近景再次点击主岛，或直接点击已显现伴岛，才进入岛内。
5. 搜索属于已知目标导航，可绕过探索步骤直接靠岸。

## 视觉语言

- 继续使用暖纸、矿物色、软丘海岸、界画建筑与正交伪 3D，不增加自由旋转真 3D 相机。
- 主岛增加低调的制图锚环，不改变岛体尺寸，避免制造价值排名。
- 真实岛间 current 在中近景成为有升降弧度的空路；高度差越大，航线抬升越明显。
- 伴岛在清晰显现前退出命中测试，避免低透明对象抢夺主岛点击。

## 响应式

- 桌面：Pixi 连续镜头自动控制伴岛披露，层级仪表显示当前尺度和已显现数量。
- 移动：初始只显示导航主岛；触摸主岛展开该群岛的伴岛。列表孪生体始终列出全部真实岛。
- 搜索会显式揭示匹配伴岛；空层筛选继续作用于主岛和伴岛。
- `prefers-reduced-motion` 保留快速落点反馈，把镜头飞行压缩到 140ms。

## 当前规模边界

- 默认世界没有为了密度混入 synthetic 内容。
- `?atlas=pixi&n=200` 与 `?atlas=pixi&n=700` 继续作为诚实标记的规模测试入口。
- 当真实岛屿增长时，同一契约可直接承接；远景不会同时显示全部岛名。

## 浏览器验收

- 桌面 1440×900：首次点击主岛留在图集并展开伴岛；第二次点击进入对应 L1。
- 搜索“组合式科学建模”：直接飞入并进入对应 L1。
- 低空筛选：可见岛全部属于低空层。
- 移动 390×844：初始 8 个导航主岛；点“生物计算的热力学代价界”后显示该组 4 个伴岛，列表仍保留 27 座岛。
- reduced-motion：首次展开在 140ms 镜头反馈后停留于图集，伴岛可操作。
