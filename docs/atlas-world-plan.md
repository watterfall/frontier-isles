# 世界地图计划 — 让规模可读 (The World Map: cartographic scale)

_2026-07-11。回应用户核心诉求:**很多领域、很多问题(几百→上千)时,如何多层次呈现,像游戏世界地图一样读得出来。有规模效应才有交叉碰撞、才有创新涌现;呈现方式若不解决,后面重构来不及。** 参照 https://xfrontier.science/(4 宏领域 → 53 命名聚类 → 1481 问题 + 9 维打分)。_

_定位:这是 `INFO-HIERARCHY.md`(语义 LOD 原理)+ `depth-plan-v2 §4`(archipelago/climate)的**执行升级 + 收口**,不是新发明。它把「原理」变成「默认路径上的 4 层世界」,并把已建但藏在 `?atlas=pixi` 后的语义缩放引擎**转正为默认体验**。_

---

## 0. 诊断:为什么现在"没有层次"

1. **解决规模的引擎已建好,但藏在 dev flag 后。** L0 语义缩放图集(Pixi 相机 + 远/中/近三层 LOD + 名牌去碰撞 + archipelago 槽,`atlas-stage.ts`/`atlas-lod.ts`,727 岛实测 1–1.6ms)只在 `?atlas=pixi` 出现。**默认打开产品走的仍是固定 1440×900 SVG 散点(~30 岛天花板)** —— 用户每次看到的就是这张扁平散点。**这是"没有层次"的第一主因,也是最大且最便宜的改进:转正。**
2. **缺最顶层:大陆/气候。** xfrontier 远景读作 4 块命名大陆;我们的远景只有 archipelago 软色块,**4 领域没有作为命名疆域 + 气候流形被画出来**(depth-plan-v2 §4 spec 了,没渲染)。"很多领域"因此不成"世界"。
3. **中层太薄。** 我们 ≤20 计算聚类、名字派生自主导域;xfrontier 证明可读中层是 **~53 命名区域**。规模一上来,20 个泛名色块 vs 53 个有性格的命名区域,后者才有"地方感"(sense of place)。
4. **下钻是硬切,不是连续。** 现在 L0(SVG)→ L1(Pixi 场景)是两个世界硬切换。游戏世界地图之所以"有层次",是因为**缩放是连续的制图下钻**(世界→区域→地方→里面),每层揭示下一层,层次是"缩"出来的、被感到的,不是三张贴图并列。
5. **每问题编码太薄。** 一岛=域色+生长态+潮汐;xfrontier 每问题是 9 维向量。"更丰富"= 在恰当 LOD 把更多**已有数据通道**(潮汐 N=A−D、休眠、离群、成员、账本活跃)映到视觉,而**不是加装饰、不发明新几何**。

## 1. 第一性原理(判每个新增)

沿用 architecture §7 + INFO-HIERARCHY,针对"规模"补三条:

1. **规模的解药不是"画更多",是"按尺度分层"**(制图泛化 / cartographic generalization)。任何尺度都不试图显示全部;显示当前尺度该有的那一层。1481 点 → 眼睛先读到 ~50 个命名区域(Koster:大脑把熟练内容压成 chunk;地图应**预 chunk**),而不是解析 1481 个点。
2. **层次 = 连续制图下钻**。世界→区域→岛→站是**一台相机的一次连续缩放**,不是场景硬切。层与层之间用制图泛化交叉淡入(远景的区域名淡出、岛名淡入),下钻叙事被"感到"。(这正是原型底注写的 **"语义缩放 L0 → 点岛入 L1"** —— 原型自己的模型,我们只是把它做深。)
3. **导航靠命名区域 + 地标 + 好奇缺口,不靠计分**(游戏地图 legibility = Lynch 的 districts/landmarks/edges/paths;Schell Lens #6 好奇、#55 可见进展)。远方雾化的区域 + 发光的灯塔离群岛 = "那边有东西" → 下钻 → 发现问题 → 读到证据。**探索动机来自看得见的未知,绝不给探究核心计分**(no-XP 不变量 + 过度辩护效应)。**移除测试**:拿掉世界地图层,"读科研前沿"本身仍成立。

**红线(design-quality 记忆 + 用户否决过的违例)**:原型是唯一视觉权威。世界地图的**结构**(大陆/命名区域/语义缩放)来自游戏地图 + xfrontier;但**视觉语汇**必须是原型的 —— 平远界画式海图、水彩软色块、域色填充、发光离群、曲线为桥。**软色块聚类,不是多边形疆界;还是那族软丘岛,只是被聚合。禁止任何棱角/多边形几何语言。**

## 2. 四层世界模型(制图下钻)

一台相机,一段连续缩放。每层"回答一个不同的问题",交叉淡入。

| 层 | 缩放 | 回答 | 画什么(原型语汇) | 数据源(全是投影,inv 14) |
|---|---|---|---|---|
| **T0 · 世界/大陆** | 最远 | 前沿的整体形貌;哪些领域在场、彼此怎么流动 | 4 领域 = 4 块**命名气候疆域**(软色washes,域色),疆域间**洋流**(桥/fork/移栽 reduce),远缘**雾化**(未探索);顶部一行"前沿天气"(总活跃) | 域流形(depth-plan-v2 §4 climate)/ currents / 账本聚合 |
| **T1 · 群岛/区域** | 中远 | 这片区域在做什么;有哪些子领域簇 | **命名区域**软色块(~30–50,不是 20 泛名);区域名 + 岛数 + 区域"体温"(聚合潮汐);几座**灯塔离群**发光浮出 | projectArchipelagos(计算)+ 可选 curated 命名/边界(place plane 编辑内容,同岛名) |
| **T2 · 岛** | 中近 | 这座岛(问题)长什么样、什么态 | 单岛:软丘海岸线(域色+hash)、生长态剪影(stage)、潮汐辉光(N=A−D)、休眠褪色、resolved 灯塔;**billboard 岛名去碰撞** | 生长/潮汐/休眠投影(projections.ts,已建) |
| **T3 · 站点/工件** | 最近(入岛) | 岛内部:谁在做、做了什么 | L1 场景(现有):站点、居民、账本仪式 | 现有 L1 |

**下钻连续性是本计划的载重梁**:T0→T2 是同一台 Pixi 海图相机的连续缩放(交叉淡入,复用已建 3-tier LOD);T2→T3(入岛)是现有 sail 动画。**默认路径 = 这张连续世界地图,不再是扁平 SVG。**

**中层区域模型(唯一需要拍板的结构分叉,先按 hybrid 默认执行)**:
- **计算聚类保底**(projectArchipelagos,纯投影,任意 N 都成立,inv 14/15)——规模安全网。
- **可选 curated 命名 + 边界叠加**(place plane 编辑数据,和岛名/居民名同性质,inv 9 可译层豁免)——给高价值区域"地方感"(如 xfrontier 的「无知测绘·盲区科学」)。
- 二者 hybrid:没有 curated 名的区域自动派生名,有的用 curated;**区域是 reduce 出来的,curated 只贴名/微调边界,不新增"画一条边界"的动词**。这样既有 xfrontier 的可读性,又守住 inv 14/15。

## 3. 显示 vs 功能平衡

- **本计划的功能环**:每层闭合一次"看见→下钻→发现"。点大陆→飞入其区域;点区域→飞入其岛;点岛→sail 入 L1。这就是"像玩游戏探索一样探索科研问题"的核心环 —— 好奇缺口(雾+远方灯塔)驱动,不计分。
- **展示先行但绑真数据**:每个视觉通道直接消费 projections/currents,后补交互动作时零数据层重构。
- **参与感顺序不倒序**:先让"看的人"被世界感 + 好奇缺口吸引(T0/T1),再让"动的人"被看见(仪式/夜班,已建),最后多人氛围。

## 4. 泳道拆分(并行,worktree 隔离)

原则:**每条泳道必须有"默认路径可感"的交付**(design-quality 记忆);flag/demo 只是中转。opus 担纲视觉权威/taste 重的层,sonnet 担纲 spec 明确的引擎/数据层。契约先行:核心投影/接口我先定,再派消费方。

| 泳道 | 模型 | 交付物 | 关键约束 |
|---|---|---|---|
| **W1 · 转正地基**(headline) | sonnet | L0 默认路径换成语义缩放图集:`ChartScreen`/默认 L0 组件挂 `AtlasStage`(现藏在 `?atlas=pixi`)。真数据(26 岛 + 生成)喂入;保留 SVG L0 fallback(服务器缺席仍渲染)。连续相机:入岛 sail 动画从图集缩放接上 | 不改 L1;默认路径必须比现在**明显**更有层次;无回退(renderMs) |
| **W2 · 大陆/气候层** | opus | T0:4 领域命名气候疆域(软色washes,域流形)+ 洋流(currents reduce)+ 远缘雾化 + "前沿天气"读数。**原型视觉语汇,软色块非多边形** | 每块疆域可答"转写哪个数据";雾=聚焦非墙;禁发明几何 |
| **W3 · 命名区域中层** | opus | T1:区域从 ≤20 泛名 → ~30–50 有性格命名区;hybrid 命名(计算派生 + curated place-plane 叠加);区域"体温"聚合;灯塔离群浮出 | curated 名走 place plane 编辑内容;区域是 reduce,不加"画边界"动词(inv 15) |
| **W4 · 尺度数据脊**(契约先行,我起草) | sonnet | 把 xfrontier 式规模喂进来验证:生成 N=200/700 真实性前沿问题(不是 `fake-*`,是可信的领域×子域×问题三元组),驱动 W1–W3 的规模测试;per-岛更多数据通道(潮汐辉光/休眠/成员)在 T2 显式化 | 生成数据须过"陌生人可信"(不是 lorem);数据通道映射不发明新形状 |
| **W5 · 探索环 + 寻路** | sonnet | 好奇缺口闭环:远方灯塔 glow、区域雾化聚焦、点大陆/区域/岛的连续下钻导航;"任一岛 ≤3 级缩放内可寻得"(INFO-HIERARCHY §6 验收);搜索/chips 接下钻 | 不计分;移除测试;下钻连续非硬切 |

**依赖**:W4 数据脊 + W1 地基先行(W2/W3 需要真规模数据 + 相机);W2/W3/W5 可在 W1 落地后并行。第一批派 **W4(契约+数据)+ W1(转正地基)**;二批 W2+W3+W5。

## 5. 验收(可证伪,扩 INFO-HIERARCHY §6)

- [ ] **默认路径**(非 `?atlas=pixi`)打开即语义缩放世界地图;700 座远景读作 ≤50 个命名区域 + 4 命名大陆,**不是散点海**。
- [ ] 连续制图下钻:世界→区域→岛→站,无硬切黑屏;任一岛 ≤3 级缩放可寻得。
- [ ] 4 领域在 T0 作为命名气候疆域 + 洋流可见,每块可回溯数据(inv 14)。
- [ ] 中层 ~30–50 命名区域,比现 20 泛名"有地方感";离群灯塔仍浮出。
- [ ] N=700 规模下 renderMs 无明显回退(现 demo 1–1.6ms);名牌任何缩放可读、无不可读重叠。
- [ ] 每层信息量与缩放匹配:远无文字过载,近无信息饥饿。
- [ ] 移除测试:拿掉世界地图层,读前沿本身仍成立(no-XP / 过度辩护)。
- [ ] 原型对照:世界地图各层用原型视觉语汇(软色块/域色/发光/曲线桥),无棱角多边形几何(screenshot 对照)。
