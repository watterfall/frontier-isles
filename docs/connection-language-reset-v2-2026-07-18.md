# 跨领域对照 v2：先看真实问题，再看系统分类

日期：2026-07-18
状态：已实现并验证

## 1. 失败点

“这些问题正在共享什么？”仍然要求用户先接受平台发明的观察方式。首屏连续出现“连接场、汇聚、共同机制、数学与桥接、方法迁移、地图线型、直接关系、w1”等词，真正的研究问题、可借用内容、差异和检验反而处在第二层。

这不是词汇不够优美，而是主语错了：界面在解释自己的图谱，而不是帮助研究者使用另一项研究。

## 2. 唯一主任务

用户只需要完成一件事：

> 选一个真实问题或一条已有对照，判断另一项研究能带来什么、哪里不能照搬、下一步怎么验证。

全局融合图继续存在，让人不必先选问题也能看见跨领域联系；但图谱类型不再作为理解这些联系的前置课程。

## 3. 首屏信息顺序

1. **标题：** 从别的研究里找办法。
2. **入口：** 先选一个正在研究的问题，或直接打开已有对照。
3. **多问题记录：** 先显示具体的共同陈述，再列出涉及的研究。
4. **两问题记录：** 先显示两项研究名称，再用一句普通话说明当前判断。
5. **筛选：** 五个类型退到一个原生折叠控件中；类型仍控制地图和列表，但不占据首屏。

## 4. 固定词汇

| 旧表达 | v2 主表达 |
|---|---|
| 问题群岛 · 连接场 | 已有研究记录 |
| 这些问题正在共享什么？ | 从别的研究里找办法 |
| 从一个具体问题进入 | 先选一个你正在研究的问题 |
| 多问题汇聚 | 同一种办法，用在不同问题上 |
| 直接关系 | 两项研究之间的具体判断 |
| 共同核心 | 能借用什么 |
| 在这个问题里如何出现 | 在这项研究里具体是什么 |
| 重要差异 / 类比边界 | 哪里不能照搬 |
| 怎样检验 | 怎么验证 |
| 地图线型 | 筛选这些对照 |
| 连接档案 | 这条判断依据什么 |
| 原论断 / 材料 | 原材料说了什么 |
| 跨域回应 | 支持或反对的理由 |
| 什么结果会改变判断 | 什么结果会让这条判断站不住 |
| 写入连接档案 | 保存这条判断 |

“机制、方程、证据、反证、复现、账本、RO-Crate、sha256”只有在它们指向真实科学对象、证据动作或溯源格式时保留；平台自造的分类名不再承担导航主语。

## 5. 两类全局行

### 多问题

- 主标题：真实的 `structure.statement`。
- 次行：参与问题的名称，以 `·` 分隔。
- 末行：`3 项研究 · 6 条记录 · 查看哪里相同、哪里不同`。
- `structure.title` 只在详情中作为记录名称或来源说明出现。

### 两问题

- 主标题：`问题 A 与 问题 B`（英文使用 `A and B`）。
- 次行按真实类型生成普通句子：
  - mathematical：`两边都用到：{真实方程/形式名称}`；
  - bridge：`有人提出，两项研究可以互相借用`；
  - evidence：`另一项研究提供了支持材料`；
  - contradiction：`两项研究的材料得出不同判断`；
  - lineage：`一个领域的方法在另一个领域被继续使用`。
- 末行显示真实记录数量与验证状态，不显示 `wN`、`ledger` 或 `curated-math`。

## 6. 详情阅读顺序

- 多问题：共同陈述 → 每项研究里的具体对应 → 不能照搬处 → 检验方法 → 来源与操作。
- 两问题：两项真实问题 → 共同使用/当前判断 → 判断依据 → 不能照搬处。
- 支持/反对表单：回应哪条材料 → 支持还是反对 → 具体理由 → 什么结果会改变判断 → 证据来源。

中英文使用同一信息顺序。已创作的中文或英文研究正文保持原语言，不自动翻译。

## 7. 不改变的底层事实

- 不改 `ConnectionChannel`、`ConnectionPathKind`、稳定 id、图层线型或账本动作。
- 不从主题、距离或聚类生成新边。
- 不把普通语言改写成大模型推断；所有具体陈述仍来自已有 mapping、current、claim 和 evidence。
- 不删除高级筛选、原始 ref、来源、证据或挑战写入，只调整出现顺序。
- 移动端继续只读，但必须使用同一套普通语言。

## 8. 验收

- 首屏不再出现“连接场、汇聚、地图线型、直接关系、跨域桥接、方法谱系、w1”等前置词。
- 不操作筛选时，用户能直接读到真实问题名称和具体判断。
- 所有类型仍可通过一个折叠筛选进入，地图过滤行为不变。
- 详情能连续回答“能借什么、哪里不能照搬、怎么验证”。
- 桌面和移动端词汇一致；中文与英文结构一致。
- 现有图谱、账本、挑战写入、键盘、响应式和 E2E 行为保持通过。

## 9. 实现与验证记录

- 桌面端 1440×1000：首屏直接显示 4 组具体共同做法和 10 条两研究判断；无“连接场、汇聚、地图线型、wN”。
- 原生筛选折叠默认关闭，键盘 Enter 可展开 5 类过滤；“支持或反对的材料”能把列表收窄到 4 条。
- 从“虚拟细胞”搜索可直接进入具体对照；从全局对照可进入两项真实研究的岛内工作台。
- 岛内工作台不再显示摆渡人、三步仪式、共同核心或机制映射；直接要求对象对应、不能照搬处和验证结果。
- 英文与中文信息结构一致；390×844 移动端文档宽度等于视口宽度，横向溢出为 0，仍保持只读边界。
- `prefers-reduced-motion: reduce` 实际生效；页面无运行时错误。
- 仓库门禁：698/698 测试（OPP 21、assets 53、renderer 114、scout 19、core 170、Web 238、server 83）；8 个工作区类型检查；生产构建；Chromium E2E 1/1；`git diff --check`。

底层 `ConnectionChannel` / `ConnectionPathKind`、稳定 id、图层线型、账本动作和写入边界未改变。保留的“方程、证据、复现、RO-Crate”只在它们真正指向科学对象或溯源行为时出现。
