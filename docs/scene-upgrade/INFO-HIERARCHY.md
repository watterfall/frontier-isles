# 信息层级规划 — Semantic LOD across scales（众岛信息呈现）

> 起因：用户指出「岛屿的丰富度和层次要加强;问题不只在进岛后,**岛屿多起来时 L0 的信息呈现也难**」。
> 本文把这个问题串成一条可执行的原理线,并说明它与已落地件、既有 ROADMAP 条目的关系。
> 定位:P3 atlas track(ROADMAP Phase C)的展开,**是对既有计划的统一 + 收紧,不是新发明**。

---

## 0. 一句话原理

**信息呈现 = f(缩放, 局部密度, variance-select 优先级)。** 任何尺度下都不试图一次显示全部;
而是**按当前缩放与密度,显示恰当"层级"的信息**——这就是语义 LOD(level-of-detail),且是**语义的**
(绑数据/意义)而非纯几何的。刚落地的 **L1 名牌 LOD 就是这条原理的最小实证**(远→印章单字,近→全名,
billboard 恒定字号=任何缩放都可读)。众岛问题 = 把同一条原理放大到 L0 图集尺度。

## 1. 问题(众岛时为何难)

现状 L0 海图 = 固定 1440×900 SVG、手放坐标、**无 zoom/pan、无 LOD/剔除**;27 座就已重叠,天花板 ~30 多座
(ROADMAP §3.11)。岛一多:名牌互相压、700 座读成一堆散点、找不到也聚不了焦。**根因不是"画得不够",
是"没有按尺度分层"**——所有信息always-on 平铺,必然过载(这也正是 v2 海即数据关系层被隐藏的同一病:§3.10)。

## 2. 三尺度 × 三层级(制图式缩放:图集→岛→站)

| 尺度 | 远(overview) | 中(region) | 近(detail) |
|---|---|---|---|
| **L0 图集(众岛)** | 不画 700 点:画 **~15–20 个群岛区**(archipelago 软色块)+ 几座**灯塔离群岛**(variance-select 发光浮出);只标群岛名 | 群岛化开为单岛(海岸线+域色),**billboard 岛名 + 去碰撞**(按 variance/活跃度定优先级,低优先冲突时降级为点) | 单岛全貌:名/域/潮汐/生长态剪影/站点数——逼近 L1 |
| **L1 岛内** | (已落地)名牌=印章单字 | (已落地)名牌=全名 | 全名+副信息;+ 密度环带/Landmark(M4.4/4.5 待做) |
| **L2 站点/工件** | 站点印章 | 站点名 | 工件卡(Z2,计划留接口) |

**连接件 = 同一个 billboard 名牌层**:恒定字号(永远可读)+ 碰撞消解(优先级高者胜)+ 分层(远/密→紧凑形,近/疏→完整形)。
**L1 已建的 label 层就是这个件**;L0 只需把它 + 去碰撞 + 群岛聚合复用上去。层级之间靠**连续的制图式下钻**(atlas→island 已有 sail 动画)felt 出来,而不是硬切。

## 3. 需要的基础设施(多数已在 §3.11/depth-plan-v2 点名,这里统一)

1. **L0 zoom/pan 相机**(前置;现固定 SVG 无相机)——L0 换 PixiJS 引擎(已建、缩略图已通)使其自然。
2. **确定性 de-overlap 布局**(已起步,近期 declutter commit)。
3. **语义 LOD 制图泛化**:3 层(远/中/近)决定画什么 + **variance-select 剔除**(离群岛永远浮在 bulk 之上,复用既有 sea-math variance-select)。
4. **群岛聚类**(archipelago:depth-plan-v2 §4,sea-math 已建**未渲染**)——密集洋流/域簇自动成 ~20 命名区。
5. **billboard 去碰撞名牌**(扩 L1 的 label 层:加冲突检测 + 优先级降级)。
6. **My Harbor 我的港湾**默认入口 + **雾化**聚焦(depth-plan-v1;不冷开整张图)。

## 4. 丰富度 & 层次(用户第二点)

- **层次(hierarchy)= 图集→岛→站的下钻叙事**。现 L0(SVG)与 L1(Pixi)是两个硬切的世界;强化 = 让缩放成为**连续制图下钻**,每层揭示下一层(LOD),层次才被"感到"。
- **单岛丰富度**:scene-upgrade 的 M4.4 Landmark×4、M4.5 密度环带、M8 粒子/夜市——均 deferred,是 per-岛richness 的下一批。
- **跨尺度呼应**:一座岛的 L0 外观应**预告**其 L1 丰富度(school 态比 empty 态在 L0 更密/更高)——生长投影已部分驱动,可显式化。

## 5. 分期(P3 atlas,Phase C;在当前 L1 场景升级收口后)

- **C1 · 相机地基**:L0 换 PixiJS + zoom/pan 相机 + de-overlap 布局。(无此则一切 LOD 无处施展)
- **C2 · 语义 LOD**:3 层制图泛化 + variance-select 剔除 + **billboard 去碰撞名牌(复用 L1 label 层)**。
- **C3 · 聚合与聚焦**:archipelago 聚类渲染 + My Harbor 入口 + 雾化。
- (关系层 rework §3.10 在 C2 之后并入:洋流/漩涡等按"on-demand lens"而非 always-on,与 LOD 同源。)

## 6. 验收(每条可证伪)

- 700 座在远景 **读成 ≤20 个命名群岛,非散点海**;离群岛仍浮出。
- 任一岛 **≤3 级缩放内可寻得**(图集→群岛→岛)。
- 名牌**任何缩放可读、且不出现不可读的重叠**(去碰撞生效)。
- 每层信息量与缩放匹配:远景无文字过载,近景无信息饥饿。
- 每个群岛/名牌**可回溯到数据**(域向量/账本;invariant 14——LOD 不是装饰,是数据的不同分辨率)。
