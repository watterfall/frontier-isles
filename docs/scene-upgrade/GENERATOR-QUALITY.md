# GENERATOR-QUALITY — 生成岛质量契约（图灵验收）

> **动因（用户 verbatim）**："点击进入岛屿还是原来样板的岛屿比较完善，有比较强的设计和
> 完整性，其他的（生成岛）设计非常粗糙，看不出来有什么改进。需要解决核心问题，要不后面
> 建岛和问题增加都无法应对。"
>
> **核心判定**：平台**所有未来内容**（建岛 / 新问题 / 采纳 / fork）都走**生成管线**。样板岛
> 「AI 之问」是手工 SVG（`apps/web/src/scene/Scene.tsx`，原型直接移植）；生成岛走 Pixi
> （`packages/renderer/src/pixi/scene-stage.ts` ← `apps/web/src/scene/layout.ts`）。二者的
> **设计质量天花板差**就是规模化的天花板：不弥合，规模化 = 粗糙化。本契约把"弥合"变成
> **可证伪的图灵验收**，并锁定"手工模板 × 数据参数化"的手法，禁止自由程序生成。
>
> 视觉唯一权威：`design/handoff/问题群岛-原型 v3.dc.html`（渲染效果为准）。质量基准：样板岛
> L1。本文件是 `M4-DESIGN.md` 的**质量护栏**，不改其数据绑定，只约束"长在更好的地面上"。

## 0. 验收总则 = 图灵测试

> **一个陌生人看生成岛与样板岛的两张 L1 截图，说不出哪张是手工做的。**

不可证伪的"更好看"不算验收。下面把它拆成 **8 条可证伪判据**（截图即可判真伪，无需读码）。
每条标注归属分期（P1/P2/P3）。合并任何一期，其判据子集必须全绿且不回归前期判据。

| # | 判据（可证伪） | 期 |
|---|---|---|
| T1 | 生成岛地面是**一块有机轮廓的连续地面**：外轮廓为平滑闭合曲线，**放大后看不到任何菱形瓦片台阶 / 锯齿海岸**。 | **P1** |
| T2 | 陆地与海面之间有**清晰的岸线过渡**（暖沙滩带 + 一条界定海岸的细线），岛屿读作"浮在海上的一块陆地"，不与海同色糊在一起。 | **P1** |
| T3 | 地形高差读作**柔和梯台 / 等高线**（连续色阶 + 细等高线），**没有逐瓦片的硬竖直崖面**。 | **P1** |
| T4 | 纸面色调统一暖调（与样板同族），**不是**冷灰绿瓦片色块拼贴；屋顶承担领域色，地面不抢色。 | **P1** |
| T5 | **任意两个相邻/成组站点之间有小径或视觉连接组织**（小径 / 院落 / 围栏 / 虚线分区），站点不再"漂"在空地上。 | P2 |
| T6 | 岛上有**居民小人**（≥ members 数的一部分）站在站点旁，提供尺度感与生活气；数量绑 `memberships`。 | P2 |
| T7 | 构图有**留白节奏与视觉重心**：非均匀铺满，有岛心锚点（Landmark/主院）+ 中环 + 稀疏岸线；空地不是均匀网格。 | P3 |
| T8 | 站点布局来自**设计师级模板**（archetype）而非随机散布：同 stage 的两座岛布局"同族但不雷同"，能一眼看出被"设计过"。 | P3 |

**回归护栏**：每期 PR 附"同角度 Miyake 前后对比截图"（本仓 `docs/scene-upgrade/`）。既有
`layout/scene` 测试**全绿**（渲染层改动，数据层不动）；`pnpm -r test && -r typecheck` 全绿。

## 1. 手法约束 —— 手工模板 × 数据参数化（designed variance，**非** random variance）

生成岛之所以粗糙，根因是**自由程序生成**：radial+fbm 高度场逐瓦片量化 → 台阶；站点按环带
概率散布 → 无组织。程序生成给的是 **random variance**（每次不同但都平庸）。样板岛的强，是
**designed variance**：少量**设计师亲手排的布局原型**，由数据**选择与填充**。

**铁律**：
1. **禁止**新增"自由画一笔"的生成（no free-form procedural terrain silhouettes / no random scatter of stations）。变化来自**在有限 archetype 上按数据插值/镜像/缩放**，不来自 rng 的自由度。
2. 每个视觉产物**仍绑既有数据**（M4-DESIGN 不变）：石碑=claim、地标=biome、萤火=active、
   建筑生长=`projectClaimState`、散木=driftwood、渡口=leavability。archetype 只决定**排布**，
   不发明新语义、不加新 verb（合架构 §7 + depth-plan §14–16）。
3. rng 只用于**受控抖动**（P7：色相 ±5% / 镜像 / 缩放 0.85–1.15 / tile 指纹），种子 = slug，
   确定性（同岛每客户端同像素）。

### 1a. Archetype 目录（草案，≥3；由 stage × 域 × 事件量选择）

生长阶段（架构 §）：**hut**（首个 artifact）→ **academy**（≥3 站点）→ **school**（publish/fork）。
每 archetype 是一张**手排布局模板**：站点槽位（slot）+ 小径骨架 + 院落/围栏 + 居民锚点 + 留白。
数据填充：站点种类→槽位映射，claim 数→建筑生长，members→居民数，域→色板/Landmark/植被。

**A1 · hut 期「小聚落」（1–2 站点，草棚初生）**
少即是。一个主屋 + 一条通向渡口的小径，大量留白 = 荒野新生。岛心留给一株树/一块碑。
```
        · · ·  (留白/散布植被)
          🌲
   ┌────────────┐
   │   [问题墙]   │        · 单主屋偏岛心
   │      │       │        · 一条小径 主屋→渡口
   │      ·       │        · 1 居民 站主屋旁
   └──────┼──────┘        · 围栏=轻，或无（未成院）
          ╎ (小径)
        ▟ 渡口
```

**A2 · academy 期「书院」（3–5 站点，合院雏形）**
样板岛就是此型。**一进院落**：主屋居上位，两厢对置，中庭留白（可置水塘/对外视图），小径成
"井"字连接，外围一圈围栏成院。居民 3–5 散在各站。
```
   ┌───────────────────────────┐  ← 围栏（院墙）
   │   [文献阁]········[问题墙]   │
   │      │      🌳       │      │  · 主屋(文献阁)居上位
   │      ├──── (中庭) ────┤      │  · 两厢对置，中庭留白+塘
   │   [白板厅]  ~塘~  [数据台]   │  · 小径"井"字连四厢
   │      │      🧍       │      │  · 居民 ♟ 散布，尺度感
   │   [实验坊]········[茶寮]     │
   └───────────╎───────────────┘
              ▟ 渡口 + 散木园(虚线)
```

**A3 · school 期「环院」（6+ 站点，多院环列 + 岛心地标）**
成规模。**岛心 Landmark**（域地标 2–3× 体量）作视觉重心，站点**环列**成 2–3 个小院，院间小径
成放射/环形骨架，密度梯度：岛心密→岸线疏。居民多，主路上可见"通勤"。
```
              [问题墙]···[展厅]
             ╱     院A      ╲
   [文献阁]──┤    ◆ Landmark  ├──[实验坊]   · Landmark 居岛心=重心
             ╲    (岛心密)    ╱             · 站点环列成 2–3 院
             [白板厅]···[数据台]            · 放射小径连院，环路收边
        院B ╎          ╎ 院C                · 岸线稀疏：礁石/灯塔/浮木
       ▟渡口   ·散布·   🌲🌲
```

> 更多 archetype（离群岛「高方差」冷色布局、休眠岛「苔藓与雾」低饱和布局）后续按域/状态扩。
> 每个 archetype 落为 `apps/web/src/scene/layout.ts` 的一张 slot 表 + 小径骨架，纯 TS、确定性。

## 2. 分期路线（每期一张图灵验收子集）

### P1 · 地面统一（**本刀，已实现**）→ 验收 T1–T4
- **做法**：`scene-stage.ts` 新增 `buildGround()`，把**逐瓦片菱形**替换为**一块连续地面**：
  瓦片并集外轮廓 → 边界追踪（`terrainBoundaryLoops`）→ Chaikin 平滑（`chaikinClosed`）→ 单
  多边形填充；高差按 level 各出一层**平滑梯台**（柔和色阶 + 细等高线 + 一条随轮廓的柔和裙边），
  **无逐瓦片崖面**；岸线暖沙滩带描边 + 界海细线 + 短裙边给"抬升的陆地"厚度；`o.tint` 指纹
  以**内部瓦片无缝 mottle**保留（只画四邻俱全的内部瓦片，不溢出平滑海岸）。
- **数据不动**：瓦片高度/命中/`elevation/shore/tint` 语义全在 layout，未改；渲染层独改。
- **验收证据**（Miyake 同角度）：`genquality-p1-ground-before.png`（改前：菱形瓦片台阶 +
  锯齿海岸 + 冷灰绿瓦片块）vs `genquality-p1-ground-after.png`（改后：连续有机地面 + 平滑
  海岸 + 暖沙滩带 + 柔和梯台）；质量基准 `genquality-sample-baseline.png`（样板岛）。既有
  443 测试全绿。
- **已知残留**（转 P3）：抬升台地内部轮廓仍偏方正（源于 elevation 区块本身方正，Chaikin 只
  能圆化到一定程度）——属"构图"层问题，P3 archetype 让高差长得更有机时一并收。

### P2 · 连接组织（小径 / 围栏 / 居民）→ 验收 T5–T6
- **小径**：沿站点对之间**实际使用关系**（同岛 claim 引用 / 站点共现）铺点线小径；数据绑
  ledger，不是装饰。骨架来自 archetype，粗细/显隐绑使用频次。
- **围栏 / 院落**：archetype 给院墙轮廓；散木园虚线分区已存在，扩为通用"院"。
- **居民小人**：数量 = `memberships`（人）+ AI 居民（capability_grants），站点旁锚点由
  archetype 槽位定；P7 抖动避免克隆感。复用样板岛的小人画法语言。
- **工作量估计**：**中（约 2–3 个工作单元）**。小径/围栏是 layout 出骨架 + renderer 画线，
  纯参数化；居民需一套 Pixi 小人 draw（可移植样板 SVG）+ members 投影接线。风险点：小径的
  "使用关系"投影要落一个 `projectStationLinks`（core，纯 reduce，无新 verb）。

### P3 · 构图 archetype（模板化布局）→ 验收 T7–T8
- **做法**：把 §1a 的 archetype 落成 layout 的 slot 表；`generator.ts` 按 stage×域×事件量**选
  archetype 并填充**（站点→槽位、claim→生长、members→居民、域→色板/Landmark）。高差、密度
  梯度、留白、岛心重心都由模板给，取代现有环带概率散布。
- **收 P1 残留**：archetype 直接排布 elevation 区块为有机形（如"岛心高台 + 缓坡下岸"），让
  台地轮廓天然有机，Chaikin 只做收边。
- **工作量估计**：**大（约 3–5 个工作单元）**。要设计并手排 3–5 个 archetype、重写 layout 的
  站点/高度/密度放置走模板、逐 archetype 图灵回归。风险点：与既有 `layout.test.ts`/
  `generator.test.ts` 的兼容（模板化会改放置坐标，需同步更新断言，属数据层，要审慎）。

## 3. 不做什么（防走偏）
- ❌ 不引入自由程序地形/随机散布替代 archetype（违背 §1 铁律）。
- ❌ 不发明新 verb / 不加"画一条连线"工具（连接组织必须 reduce 自账本，合 depth-plan §14–15）。
- ❌ 不为炫技加动效；动态只在数据绑定处（M5/M8 既有门禁：`active` 才动）。
- ❌ 不改视觉 token 契约；`#5A6C9E`（AI ink）等组件级常量不动。
