# 可执行模型层 v1

_2026-07-18 · 产品边界、学习闭环与首个实现切片_

## 1. 决定

这部分不拆成另一个项目，但在代码上建立独立的模型子系统。

- **仍属于 Frontier Isles**：真实问题来自地图和岛屿；可迁移关系来自已有研究记录；模型运行后回到同一本考察札记。拆成另一个产品会切断这条路径。
- **不属于关系图**：关系图回答“为什么值得比较”；模型运行时回答“我能否亲手让这条规则跑起来”。图中的一条边不能自动变成一个模型，模型运行也不能自动成为研究证据。
- **不属于聊天回答**：AI 可以补数学、代码、参数解释和绘图，但学习者必须决定模型里有什么、它们如何互相影响、先做出什么预测，以及结果哪里不能外推。

因此产品形成四个连续但不同的真相层：

1. **地图 / 岛屿** — 我关心哪个真实问题；
2. **研究对照** — 别的领域有什么办法可能可用，依据和差异是什么；
3. **模型台** — 我把最小规则搭出来并运行；
4. **考察札记** — 我预测了什么、改了什么、看见了什么、哪里失效。

## 2. 第一原则

离开模型台时，学习者必须带走一个自己做过决定的模型运行记录，而不是一个播放过的动画。

一次有效运行必须同时包含：

- **放进模型的对象**；
- **对象能看到或影响什么**；
- **对象怎样更新**；
- **观察哪个整体量**；
- **运行前的预测**；
- **实际参数与结果**；
- **换到另一个领域时不能照搬的地方**。

少一项，运行可以作为演示，但不能保存为学习记录。

## 3. 交互闭环

模型台不是课程向导，也不是代码编辑器。默认页面同时保留四块可以互相校验的内容：

1. 左上选择一个具体问题或现象；
2. 左下用普通语言搭出对象、相互作用和观察量；
3. 中央直接看系统运行和整体量变化；
4. 右侧保留预测、结果、适用边界和“把同一规则换到另一个领域”。

运行顺序：

`选择现象 → 调整最少量参数 → 先预测 → 运行 / 单步 → 对照整体量 → 换领域 → 写明边界 → 保存`

数学式默认折叠。展开后必须能逐项指回页面上已经操作过的对象和参数，不出现只有公式没有可操作含义的符号。

## 4. 首批两个模型家族

### 4.1 同步

同一套有限耦合振子规则在四个解释环境中运行：

- 萤火虫闪光；
- 心脏起搏细胞；
- 观众鼓掌；
- 交流电网频率。

学习者控制：个体节奏差异、彼此影响强度。观察量是相位一致程度 `R`。四个环境共享计算内核，但分别写明边界：真实萤火虫的脉冲、心肌细胞的激发与不应期、鼓掌者的策略变化、电网的惯性与网络拓扑都没有被一阶全连接相位模型完整表示。

### 4.2 热、扩散、静电势与稳态流

四个解释环境复用同一个离散邻域算子：每个格点读取邻居，按局部差异更新。

- 热扩散与物质扩散是随时间演化的问题；
- 无源静电势和理想稳态势流是边界值问题；
- 相同的离散 Laplacian 只意味着可以复用求解结构，不意味着物理量、守恒条件和因果过程相同。

学习者控制：局部更新速率、边界/源的设置。观察量是空间不均匀程度或稳态残差。

## 5. 代码边界

模型子系统必须保持浏览器安全、确定性和可测试：

```text
models/catalog.ts       authored families, substrates, labels, sources, limits
models/oscillators.ts   pure seeded state + step + order parameter
models/scalarField.ts   pure grid state + stencil step + residual/statistics
models/types.ts         launch context and durable run receipt
components/model/*      model workspace and visual/list twins
```

约束：

- 内核不导入 React、Pixi、API、ledger 或完整岛屿数据；
- 相同 seed 与参数必须得到相同结果；
- 模型内容不能生成结构图边或研究证据；
- 首屏仍遵守 L0/L1 的延迟加载边界，模型台作为独立 chunk；
- 模型运行记录进入本地 versioned notebook，不写研究账本；
- 桌面和手机都能完成本地建模；只有需要身份和权限的研究账本写入继续保持桌面限制。

## 6. 视觉与可访问性

- 模型台是铺开的实验纸，不是卡片仪表盘：规则写在左侧，运行画面居中，预测与迁移记录在右侧。
- 同步模型同时提供相位圆/群体闪光与按编号排列的相位列表；场模型同时提供色阶网格与中心值、极差、残差的文字读数。
- 所有滑杆有实时数值、键盘步进和普通语言解释；播放之外必须有重置与单步。
- `prefers-reduced-motion` 下“运行”直接计算到结果，仍可用“单步”观察离散变化。
- 手机重新排成线性顺序，保留预测、运行、换领域和保存，不把核心学习行为删掉。

## 7. 保存与来源

每条 `ModelRunReceipt` 至少保存：

- 家族、解释环境、seed；
- 参数快照；
- 学习者预测；
- 初始与最终观察量、步数；
- 学习者确认的适用边界；
- 来源结构或问题上下文（若从研究对照进入）；
- ISO 时间。

运行记录是个人考察材料，不是验证过的科学主张。Markdown 导出必须明确这一边界，并保留模型来源链接。

## 8. v1 验收

1. 从地图可以一键进入模型台；从“耦合振子同步”的研究对照可以带着上下文进入。
2. 学习者不看公式也能说清模型里有什么、它们怎样影响彼此、自己观察什么。
3. 未做预测不能保存；未确认适用边界不能保存。
4. 同步模型改变耦合强度后，`R(t)` 的变化可见且可复现；换解释环境不偷偷改变计算规则。
5. 场模型可用同一 stencil 在四个解释环境间切换，并明确动态扩散与稳态边界值问题的差异。
6. 保存后刷新页面仍能从考察札记导出完整运行记录。
7. 桌面、390px 手机、键盘和 reduced-motion 都能完成 `预测 → 运行 → 换领域 → 保存`。

## 9. 暂不声称

- v1 不是自动发现所有跨学科同构的 AI；它先把“发现之后如何真正学会”做通。
- v1 的两个模型是可解释的教学简化，不是萤火虫、心脏、电网或连续介质的高保真数字孪生。
- v1 不把个人运行结果写成 ledger evidence，也不让模型目录凭空扩充结构图。
- 标度、相变、渗流、反馈、反应扩散等家族在同一 contract 下扩展，但只有在具备可运行内核、来源、迁移边界和学习任务后才进入主界面。
