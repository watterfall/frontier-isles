海即数据 — 海不是背景，是三条读得出的数据通道

岛浮在海上，海面每一处暗度/漩涡/岸边读数都直接转录自账本与 substrate 分，没有一处是装饰。三条通道彼此正交：①深度暗化＝领域抽象度（substrate→uDepth，col *= (1.0 - uDepth)），②undertow 漩涡＝争议度（未化解的驳斥→uUndertow，以真实量级 1-refute 为主展品，非满量程），③文字解码器＝把海重新写成一行可读文字（list-twin）。关键不变量：深≠色——下面 Ch.1 两格岛身色相恒定（数理），变的只有海，证明 depth 与 domain-hue 是两根轴（sea.ts:100–106）。

Ch.1 深度暗化 sea depth = domain abstractness

substrate 抽象度 ∈ [0,1]（0 = 浅滩/偏应用，1 = 深水/偏形式）经 seaDepthAt() 变成一个只加暗、不加色的 alpha。海面着色器把它当乘法暗化因子；离岸越远沉得越暗，且下沉速度随 uDepth 增大（抽象岛更快跌进暗海）。下面两格同为平静海(0 驳斥)，只有 substrate 不同：

// packages/renderer/src/sea.ts
export const SEA_DEPTH_MAX_ALPHA = 0.42;        // L88 · 最深水的最大暗化 alpha
export function seaDepthAt(substrate) {          // L107–111
  if (substrate == null || Number.isNaN(substrate))
    return { depth: null, overlayAlpha: 0 };  // 无分 → 不暗化(诚实缺席)
  const depth = clamp01(substrate);
  return { depth, overlayAlpha: depth * SEA_DEPTH_MAX_ALPHA };
}
// sea-mesh.ts · uDepth = seaDepthAt(substrate).overlayAlpha ∈ 0..~0.42
col = mix(col, uDeepColor, smoothstep(0.0,1.0,radial)*(0.28+0.55*uDepth)*(1.0-here)); // L129 离岸渐变，reach∝uDepth
col *= (1.0 - uDepth);                        // L154 · 就是这一乘：深＝暗，never a new hue
INVARIANT · 深≠色 depth 读的是真实 substrate 分，刻意不是 domain-hue 向量——独立的一根轴，永不是"旋转过的色相副本"（sea.ts:100–106）。substrate 为 null/NaN 时 overlayAlpha=0，宁可不画暗度也不编假深浅（honest absence beats decoration）。

岛身色相恒定 #C9D8E6（数理）；左浅滩 uDepth 0.076、右深水 uDepth 0.386 —— 同一色相、不同暗度，就是 depth ⊥ hue。

Ch.2 undertow 漩涡 disputed-sea = contention magnitude · 1-refute primary

争议度＝未化解的驳斥数。岛屏先数出被驳的 claim（ghost==='refuted'），过传递曲线成 contention，驱动着色器的 uUndertow——一块缓慢自转、把水压暗到 50% 的暗涡。主展品是真实量级 1-refute（u=0.60）：日常一条被驳，海面就该起这么大的涡，不是满量程。下面 0/1/2/3-refute 四格条带与生产 shader 同参、深度恒定（substrate 0.45），变的只有争议；u=1.0 仅作 3+ refutes 端点。

// apps/web/.../GeneratedIslandScreen.tsx (~L206–214)
const refuted = projected?.filter(c => c.ghost === 'refuted').length ?? 0;
setSeaStats({ substrate: det.object.frontier?.substrate,
  validates: events.filter(e => e.action==='validate').length,
  refutes:   events.filter(e => e.action==='refute').length,   // ← 事件计数，供解码器"驳斥"文字
  bridges:   events.filter(e => e.action.startsWith('bridge')).length,
  contention: refuted === 0 ? 0 : Math.min(1, 0.6 + (refuted-1)*0.2) });  // ★ Round 6 修订曲线
// packages/renderer/src/pixi/sea-mesh.ts (L157–159) · uUndertow: 0=calm, 1..N=contention
if (uUndertow > 0.001) {
  float swirl = smoothstep(0.50,0.82, fbm(w*0.018 - vec2(uTime*0.12, uTime*0.08)));  // ★ R6
  col = mix(col, col*0.50, swirl * uUndertow * (1.0-here)); }               // ★ R6 (was 0.62)
// scene-stage.ts (L1013–1016) setUndertow(on: boolean|number) → uUndertow = v
TRANSFER · Round 6 修订(待合并) contention = refuted===0 ? 0 : min(1, 0.6 + (refuted-1)×0.2)：0→0(平静·诚实零点)，1→0.60(主展品)，2→0.80，3→1.0(封顶)。旧 shipped 曲线是 min(1, refuted×0.5)——本卡转录的是本轮工程目标值，orchestrator gate 须对合并后代码复核。
DON'T CONFLATE undertow 强度读 refuted（claim 幽灵数），解码器"驳斥/refuted"文字读 refutes（账本事件数）——两个来源，屏上可能不同数（下面演示里我把它们设成相等，仅为对齐讲解）。

深度全格恒定 substrate 0.45（uDepth 0.189）；uUndertow 随驳斥数 0 → 0.60 → 0.80 → 1.0。金框 = 1-refute 主展品（日常量级）；最右 3-refute = 满量程端点，仅标注不主打。

Ch.3 文字解码器 the sea, rewritten as one readable line (list-twin)

海的每条视觉通道都配一份纯文字孪生（list-twin）：不靠让读者认色、认涡，而是把 substrate 与关系计数直接写成一行。逐字转录下面两个 <span>（GeneratedIslandScreen.tsx L403–404）——注意实际前缀是 ≈ 与 ⇄，不是 brief 里写的 🌊：

// L403 — 深度行：substrate 保留两位小数 + 抽象度档位
{seaStats?.substrate != null && <span>≈ {t('island.seaData.depth')} {seaStats.substrate.toFixed(2)} · {t(abstractKey(seaStats.substrate))}</span>}
// L404 — 关系行：仅非零的关系入列，' · ' 连接（0 值不出现，honest）
{relParts.length > 0 && <span>⇄ {relParts.join(' · ')}</span>}
// L315 abstractKey 档位阈值
abstractKey = s => s >= 0.66 ? '…abstract.hi' : s >= 0.33 ? '…abstract.mid' : '…abstract.lo';
