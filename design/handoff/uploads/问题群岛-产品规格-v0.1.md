# 问题群岛 Frontier Isles · 产品规格 v0.1

> 等距视角的开放科学协作平台：把前沿挑战问题变成可以居住的虚拟实验室。
> 工作名「问题群岛」，备选：格致群岛、书院群岛、散木群岛。

---

## 0. 一句话与三个命题

**一句话**：一个 Web 平台，任何人可围绕一个前沿挑战问题"建岛"（创建虚拟实验室），实验室以等距视角的空间 diorama 呈现，协作产出直接生长为岛上的建筑与器物，全部过程默认开放。

**三个设计命题**（所有功能决策以此为裁判）：

1. **研究状态即建筑** —— 实验室的阶段、活跃度、产出不用仪表盘汇报，而是直接可见为空间形态。建筑随真实贡献生长，不可购买、不可装饰性堆砌。
2. **昼夜即记录与实际** —— 白天模式 = 正式记录 D(t)（策展层：结论、文档、发表物）；夜晚模式 = 实际探索 A(t)（全量层：死胡同、废稿、直觉批注、争论现场）。昼夜切换是一等公民交互。理论出处：夜晚科学形式模型 N(t) = A(t) − D(t)。
3. **群岛即图谱** —— 宏观层是可航行的前沿问题海图。岛屿按领域聚落，跨学科关联为桥，图谱评分（九维）驱动岛屿形态；离群/高方差条目在视觉上反常发光。

## 1. 概念模型（领域对象 ↔ 空间隐喻）

| 领域对象 | 空间隐喻 | 说明 |
|---|---|---|
| 平台 | 群岛海图 | 语义缩放的最外层 |
| 虚拟实验室 | 一座岛 | 围绕一个前沿挑战问题（QFocus） |
| 研究阶段/功能 | 建筑（站） | 问题墙、数据台、白板厅、文献阁、实验坊、展厅、茶寮、散木园 |
| 工件 artifact | 器物 | 摆在瓦片上的文档/数据集/画布/笔记本，点击展开 |
| 成员（人 + AI） | 居民 | AI 智能体经 MCP 以驻岛居民身份存在 |
| 贡献历史 | 小径与年轮 | 足迹踩出 desire path；建筑基座刻事件年轮 |
| 分支研究 | Fork 出的姊妹岛 | 保留血缘链接（海图上以细线相连） |
| 跨学科链接 | 桥 | 手动搭桥 + 图谱推荐 |

**站点清单（v1 固定八种，后续可扩展）**：

- 问题墙：QFT 工作区。岛的创生地，钉问题、改写、投票、聚焦出 QFocus。
- 数据台：数据集与来源登记（链接优先，不做重型存储）。
- 白板厅：无限画布（共同编辑）。
- 文献阁：文献收藏与批注流。
- 实验坊：可运行工件（notebook 链接、仿真、原型 demo）。
- 展厅：策展层。只有这里的内容出现在"白天"的对外视图，可导出为开放实验记录本。
- 茶寮：闲聊区。刻意不与任何产出指标挂钩。
- 散木园：无考核野地。偏题发现、无用把玩、废案陈列。机制上允许"移栽"——散木园中的器物可被一键升格进正式站点，并保留"曾为散木"的标记（散木定律的可视化）。

## 2. 世界结构与导航

**三级语义缩放**：

| 层级 | 视图 | 核心操作 |
|---|---|---|
| L0 群岛 | 海图（非等距，界画式平远构图） | 航行、搜索、按学科/评分/活跃度过滤、看桥 |
| L1 岛 | 等距 diorama（2:1） | 走动/环视、进站、昼夜切换、看小径 |
| L2 台 | DOM 面板（文档/画布/数据视图） | 实际协作编辑发生地 |

- 转场语言：卷轴展开/收拢（横向 wipe + 视差），拒绝生硬的路由跳转。
- 每个空间对象都有**列表孪生体**：侧边栏永远可以按列表检索岛上一切。空间是氛围层，列表是效率层。快捷键直达任意站点。
- 移动端：L0/L1 只读浏览 + 通知；L2 完整可用。

## 3. 核心机制

### 3.1 建岛仪式（QFT 即 onboarding）
创建实验室不是填表单。流程：写下驱动性素材 → 发散提问（计时）→ 标记开放/封闭问题并改写 → 排序聚焦 → 产出 QFocus 作为岛名与问题墙首钉。整个仪式过程本身作为岛的"创世事件"永久保留在夜晚层。

### 3.2 研究即建筑（生长规则）
- 阶段驱动形态：空岛（仅问题墙）→ 草棚（首批器物）→ 书院（多站运转）→ 学派（有 fork 后代、有展厅出版物）。
- 生长只由**真实事件**触发（新器物、新成员首次贡献、展厅出版、被 fork），无任何货币/点数中介。
- 衰败可见：长期无事件的岛缓慢长出苔藓与雾——不删除、不排名羞辱，只是诚实呈现休眠。

### 3.3 昼夜机制（本平台的签名交互）
- 数据基础：事件溯源。一切写操作是 append-only 事件；"白天"是展厅策展视图，"夜晚"是全量事件流的空间化回放。
- 夜晚呈现：灯笼点亮有夜间活动的站点；废稿以半透明"魂影"器物出现在原位；争论线程以萤火虫群悬浮；可拖动时间轴回放岛的任意一夜。
- 隐私边界：夜晚层默认对岛内成员可见，岛主可选择"开放之夜"（对外公开全部过程——最激进的开放科学姿态，给徽记）。

### 3.4 散木园与移栽
见 1. 站点清单。关键指标反向设计：平台层面追踪"移栽率"（散木 → 正式产出的转化），作为面向研究者的洞察而非排行榜。

### 3.5 Fork 与合流
- Fork 一座岛 = 复制其问题墙与选定器物的引用，血缘线画在海图上。
- 合流：姊妹岛可将器物"回赠"母岛（PR 隐喻），母岛展厅标注来源。

### 3.6 小径（贡献可视化）
成员在岛上的移动与操作在地面累积为踩踏痕迹。小径即热图、即 desire path、即最诚实的"这个实验室真实的工作流长什么样"。

### 3.7 三元居民（AI via MCP）
- 每岛可入住 AI 居民，角色模板：文献斥候（扫描新文献并投递到文献阁）、魔鬼辩护人（对问题墙的聚焦结果提出反例）、综合者（周期性把夜晚层压缩为白天层草稿，供人类批准）。
- 实现：每座岛暴露一个 MCP server（读写该岛事件流与器物），AI 居民 = 挂接该 server 的 Claude 会话/Agent。人类对 AI 产出的采纳动作本身也是事件（三元互动可度量）。

### 3.8 成员进阶（对应 B0–B4，不用等级词汇）
访客（可看白天）→ 学徒（可在散木园与茶寮活动）→ 研究员（全站读写）→ 驻岛（可策展展厅）→ 山长（岛主，可开放之夜、可合流）。晋升由岛内既有成员确认，不由系统自动发放。

## 4. 反机制（明确不做）

- 不做 XP / 积分 / 徽章商店 / 全局排行榜。动机来源 = 自主（建岛自由）+ 胜任（建筑生长可见）+ 归属（小径与在场）。
- 不做强制实时。异步为默认，实时是彩蛋（同时在岛会看到彼此）。
- 不做重型文件存储（v1 一律链接 + 元数据 + 预览）。
- 不做算法信息流。发现靠海图航行与桥，不靠推荐流。
- 空间交互永不成为唯一路径（列表孪生体铁律）。

## 5. 设计方向（Design brief 详见 §9）

- **美术命题：数字界画**。等距投影是界画与长卷的数字血脉；实验室是书院园林而非太空舱。
- 色板基调：纸底（暖米白）、墨线、矿物色点缀——石青 #2E5E8C 系 / 石绿 #3E9B7E 系 / 赭石 #B5673A 系 / 藤黄点睛。夜晚层整体转靛蓝纸 + 灯笼暖光。
- 字体：中文正文用系统级高质量黑体；数字与代码用等宽；标题可探索仿宋/宋体的现代化处理。像素字体仅作点缀（坐标、年份）。
- 动效：卷轴转场、水面微波、灯笼摇曳；全部尊重 prefers-reduced-motion。
- 签名元素（唯一记忆点）：**昼夜拨杆**——一个太阳/灯笼形态的全局开关，切换时整座岛做一次光影翻转 + 魂影显隐。

## 6. 技术架构

### 6.1 结论先行
- **前端**：TypeScript + Vite + React（DOM 面板层）+ **PixiJS 8**（等距画布层）。不用 Godot：平台需一个链接直达、可嵌入、文本 UI 密集；Godot 留给学生游戏管线。
- **协同**：**Yjs**（文档 + awareness 在场）+ y-websocket（自托管）。文档层直接复用/对接「砚」（BlockNote + Yjs）。
- **后端**：Bun + TypeScript + Hono。核心存储 = **append-only 事件日志**（每岛一个流）+ SQLite（起步）/Postgres（扩展）做投影读模型。与 pm 编排器的 events.jsonl 模式同构。
- **身份**：GitHub OAuth 起步，P3 接 ORCID。
- **AI 层**：每岛一个 MCP server（工具：读事件流、写器物、发提议）。
- **部署**：单体起步（Fly.io / 自托管均可），静态资源 CDN。

### 6.2 等距渲染要点（给 Claude Code）
- 2:1 菱形瓦片，基准 128×64；世界→屏幕：`sx = (gx−gy)·64; sy = (gx+gy)·32`；拾取用逆矩阵 + 器物像素 mask。
- 深度：按锚点 (gx+gy) 排序；跨瓦片建筑切块为多精灵。
- 分块（chunk）渲染 + 视口剔除；海图层的岛屿缩略图离线预渲染为位图。
- 资产管线：建筑/器物用扁平矢量（SVG→纹理图集），保证 AI 可生成、可参数化换色（昼/夜两套色只换 palette 不换形）。

### 6.3 数据模型草案

```
User(id, handle, orcid?, avatar)
Island(id, slug, qfocus, domain_tags[], scores{9d}?, created_by, visibility, night_open:boolean, parent_island?)
Membership(user, island, role: visitor|apprentice|researcher|resident|master)
Station(id, island, kind: questions|data|canvas|library|workshop|gallery|tearoom|driftwood, pos{gx,gy}, level)
Artifact(id, island, station, type: doc|dataset|canvas|link|notebook, ref, pos, is_ghost:boolean, was_driftwood:boolean)
Event(seq, island, ts, actor(user|agent), verb, payload)   // append-only，一切之源
Bridge(island_a, island_b, note, created_by)
AgentResident(id, island, role_template, mcp_endpoint)
```

投影（读模型）：白天视图 = Gallery 策展 + 各站当前态；夜晚视图 = 事件流的时间切片。

### 6.4 与既有系统的接口
- **xfrontier.science**：选出来条目作为"待认领荒岛"播种海图；九维评分映射到岛屿形态参数（体量/海拔/光晕），离群检测结果映射为异常发光。
- **砚**：L2 文档面板即砚的嵌入或同栈实现，Typst 导出 = 展厅出版物 PDF。

## 7. 里程碑与验收

| 阶段 | 范围 | 验收标准 |
|---|---|---|
| **P0 原型** | 海图 + 一座样板岛 + 昼夜拨杆 + 器物展开面板（静态数据） | 5 分钟内让陌生人说出三个命题中的两个 |
| **P1 骨架** | 建岛仪式、等距渲染器、Yjs 在场 + 白板厅共同编辑、问题墙、事件日志、列表孪生体 | 两人异地登同一岛，可见彼此、可共同编辑、事件可回放 |
| **P2 机制** | 昼夜全量回放、建筑生长、散木园 + 移栽、Fork、首个 AI 居民（文献斥候） | 一座岛跑通"散木→移栽→展厅出版"全链 |
| **P3 接图谱** | xfrontier 播种、桥、ORCID、开放实验记录本导出、开放之夜 | 海图承载 700+ 岛不卡顿；一份展厅可导出带 DOI 引用的 PDF |

## 8. 开放决策点（待定，均不阻塞 P0/P1）

1. 命名终选（问题群岛 / 格致群岛 / 其他）。
2. 夜晚层默认可见范围（岛内 vs 完全公开）的社区规范。
3. 许可默认值（建议 CC-BY-4.0 + 代码 MIT，建岛时可改）。
4. 是否引入轻量音景（水声/夜虫）——倾向做，但放 P2。
5. 海图布局算法：领域聚类的力导向 vs 手工策展分区（建议混合）。
